# Point Zero One Digital Metrics, SLOs, and Ops Dashboards

## Overview

This document outlines the metrics, Service Level Objectives (SLOs), and operational dashboards for `pzo_verified_bragging_rights_run_explorer_v1`. The focus is on providing clear, concise, and execution-grade information to facilitate efficient monitoring, troubleshooting, and optimization of the game's performance.

## Non-negotiables

1. **Strict TypeScript**: All code adheres to strict TypeScript mode for type safety and consistency.
2. **Deterministic effects**: All game effects are designed to be deterministic, ensuring predictable behavior and reproducibility.
3. **No 'any'**: The use of the `any` type is strictly prohibited in TypeScript code to maintain type safety.
4. **Production-grade**: All metrics, SLOs, and dashboards are designed with production environments in mind, ensuring they are deployment-ready.

## Implementation Spec

### Explorer Page Metrics

- **User engagement**: Measures the time spent by users on the explorer page, providing insights into user interest and satisfaction.
- **Exploration depth**: Tracks the number of levels explored by each user, indicating the extent of their gameplay.
- **Achievements unlocked**: Records the number of achievements unlocked by users, reflecting their progress and skill level.

### Verification Pipeline Metrics

- **Verification success rate**: Monitors the percentage of successful verifications in the pipeline, indicating the reliability and efficiency of the verification process.
- **Average verification time**: Calculates the average time taken for a verification to complete, helping identify bottlenecks and optimize performance.
- **Failed verifications count**: Tracks the number of failed verifications, enabling quick identification and resolution of issues.

### Share Virality Metrics

- **Shares created**: Counts the total number of shares generated by users, reflecting the game's virality.
- **Shares accepted**: Records the number of shares accepted by other players, indicating the effectiveness of the sharing mechanism in attracting new users.

### SLOs

- **Verification success rate SLO**: A minimum of 95% to ensure a reliable and efficient verification process.
- **Average verification time SLO**: An average verification time of less than 10 seconds to maintain user engagement and satisfaction.

### Ops Dashboard Layout

The operational dashboard is divided into the following sections:

1. **Explorer page metrics**: Displays user engagement, exploration depth, and achievements unlocked.
2. **Verification pipeline metrics**: Shows verification success rate, average verification time, and failed verifications count.
3. **Share virality metrics**: Presents shares created, shares accepted, and other relevant share-related metrics.
4. **Alerts and notifications**: Displays any critical alerts or notifications related to the game's performance or user activity.
5. **Performance monitoring**: Provides real-time insights into the game's overall performance, including CPU usage, memory consumption, and network traffic.
